Build a full-stack AI photo generation service landing page. Use React 18 + Vite for frontend, Node.js + Express for backend, and PostgreSQL for the database.

=== TECH STACK ===
- Frontend: React 18, Vite, TailwindCSS, Framer Motion, React Router v6, Axios
- Backend: Node.js, Express, Multer, JWT, bcrypt, Sequelize ORM
- Database: PostgreSQL
- Image storage: Cloudinary (use environment variables for keys)

=== COLOR SCHEME & DESIGN ===
Dark theme only:

All animations via Framer Motion

=== DATABASE SCHEMA ===

Create the following PostgreSQL tables using Sequelize:

1. Table: styles
- id: UUID, primary key
- title: VARCHAR(255), not null
- short_description: TEXT
- full_description: TEXT
- prompt: TEXT
- negative_prompt: TEXT
- price: DECIMAL(10,2)
- preview_image_url: VARCHAR(500)
- example_images: JSONB (array of image URLs, minimum 3)
- generation_time: INTEGER (seconds)
- is_active: BOOLEAN, default true
- sort_order: INTEGER, default 0
- orders_count: INTEGER, default 0
- category: VARCHAR(100) (e.g. 'Портрет', 'Арт', 'Бизнес', 'Фэнтези', 'Аниме')
- created_at: TIMESTAMP

2. Table: users
- id: UUID, primary key
- email: VARCHAR(255), unique, not null
- name: VARCHAR(255)
- password_hash: VARCHAR(255)
- role: VARCHAR(50), default 'user'
- is_blocked: BOOLEAN, default false
- created_at: TIMESTAMP
- last_login: TIMESTAMP

3. Table: orders
- id: UUID, primary key
- user_id: UUID, references users(id)
- style_id: UUID, references styles(id)
- status: VARCHAR(50) (values: 'pending', 'paid', 'processing', 'completed', 'failed')
- amount: DECIMAL(10,2)
- source_photo_url: VARCHAR(500)
- result_photos: JSONB
- payment_id: VARCHAR(255)
- error_message: TEXT
- created_at: TIMESTAMP
- completed_at: TIMESTAMP

Seed the database with 6 sample styles with realistic Russian titles, descriptions and placeholder Cloudinary image URLs.

=== BACKEND API ===

Create the following REST API endpoints:

GET /api/styles — return all active styles (is_active=true), ordered by sort_order. Each style object must include: id, title, short_description, preview_image_url, example_images, price, generation_time, orders_count, category
GET /api/styles/:id — return single style by id with all fields
POST /api/auth/register — register user (email, name, password), return JWT token
POST /api/auth/login — login user, update last_login, return JWT token
GET /api/auth/me — return current user data (requires JWT)
